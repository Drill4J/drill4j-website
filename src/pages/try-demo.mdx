## 1. Clone the example repository:

```bash
git clone https://github.com/Drill4J/realworld-java-and-js-coverage
```

## 2. Start Drill4J Admin

<ListItem number={'2.1.'}>Start docker containers using the following command:</ListItem>

```bash
./demo-up.sh
```

<ListItem number={'2.2.'}>

After the installation finished go to Drill4J Admin: [http://localhost:8091](http://localhost:8091)

</ListItem>

<ListItem number={'2.3.'}>

Press “Continue as a Guest” and **register agents**

</ListItem>

<Image src={'/img/java-plus-js-agents-table.png'}/>

<ListItem number={'2.4.'}>

Click through *Step 1* and *Step 2* of the registration wizard, but make sure that Test2Code plugin is **enabled** at *Step 3*

</ListItem>

<AdditionalInformation >

**Important!** Don't forget to set target host "http://localhost:8080/" on the second step for frontend agent.

</AdditionalInformation>

<ListItem number={'2.5.'}>

Open sample application at [http://localhost:8080](http://localhost:8080)

</ListItem>

<ListItem number={'2.6.'}>

To collect coverage run tests using this command:

</ListItem>

```bash
./gradlew clean :build1:test
```

See the testing results on Test2Code plugin’s page and finish active scope to add collected coverage to your build.


## 3. Deploy additional builds

To discover Drill4J more, you have to deploy additional builds.

<ListItem number={'3.1.'}>

Deploy the **second build** of the app using this command:

</ListItem>

```bash
./demo-deploy-build2.sh
```

Wait for notification about new build arrival will apear in the notification panel and navigate to it.

<ListItem number={'3.2.'}>

To collect coverage for the second build run tests using this command:

</ListItem>

```bash
./gradlew clean :build2:test
```

Now you will be able to see the list of **Risks** (uncovered new and modified methods) and suggested **Tests to Run**

<ListItem number={'3.3.'}>

Deploy the **third build** using this command:

</ListItem>

```bash
./demo-deploy-build3.sh
```

<ListItem number={'3.4.'}>

To collect coverage for the third build run tests using this command:

</ListItem>

```bash
./gradlew clean :build3:test
```

## Also, you can down demo setup:

```bash
./demo-down.sh
```

## 1. Start Drill4J Admin

First, you need to download a special [Docker-compose file](https://github.com/Drill4J/example-configs/blob/main/demo/docker-compose.yml)

Start Drill4J using the following command in the terminal:

```bash
docker-compose up -d drill-admin admin-ui agent-files
```

## 2. Start example Application

To start the example App with Drill4J Agent use this command:

```bash
docker-compose up -d example-app
```
After that you can find **Drill Admin** on [http://localhost:8091](http://localhost:8091)

And **Application UI** is located here: [http://localhost:8087](http://localhost:8087)

Install the latest version of [Chrome Extension](/docs/installation/extension)
